# 国际化(i18n)方案

## 需求分析
- 如果仅公司内部使用，中文即可
- 如果产品化推广，需要多语言支持
- 参考APITable: 已支持多语言(英/中/日/法/德/西)

## 前端方案: next-intl / react-i18next

### react-i18next (SPA方案)
```typescript
// i18n配置
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

i18n.use(initReactI18next).init({
  resources: {
    en: { translation: require('./locales/en.json') },
    zh: { translation: require('./locales/zh.json') },
  },
  lng: 'zh',
  fallbackLng: 'en',
});

// 使用
const { t } = useTranslation();
<h1>{t('table.create')}</h1>
```

### next-intl (Next.js方案)
- 专为Next.js设计
- 支持SSR/SSG
- 路由级国际化(/en/tables, /zh/tables)

## 后端方案
- 错误消息多语言
- 邮件/通知模板多语言
- API响应消息多语言

```typescript
// 后端i18n
class I18nService {
  t(key: string, lang: string, params?: Record<string, any>): string {
    return this.translations[lang]?.[key] || key;
  }
}
```

## 数据层国际化
- 字段名/视图名等用户自定义内容 **不翻译**
- 系统UI文案翻译
- 日期/数字格式本地化

## 推荐策略
1. **初期**: 中文为主，代码层面做好i18n准备(用key而非硬编码文字)
2. **中期**: 引入react-i18next，翻译核心界面
3. **长期**: 支持用户选择语言，贡献者社区翻译
