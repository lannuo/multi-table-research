# 多维表格场景化模板设计

## 核心理念
多维表格不只是空表格——通过**预设模板**，用户可以开箱即用地管理任务、项目、客户、招聘等业务场景。模板 = 数据模型 + 视图 + 自动化 + 仪表盘 的完整封装。

---

## 二、核心场景模板设计

### 场景1: 任务管理

#### 数据模型

```
任务表 (Tasks)
├── 任务名称     [文本]
├── 状态         [单选: 待办/进行中/已完成/已取消]
├── 优先级       [单选: 紧急/高/中/低]
├── 负责人       [人员]
├── 协作人       [人员(多选)]
├── 截止日期     [日期]
├── 开始日期     [日期]
├── 所属项目     [关联 → 项目表]
├── 标签         [多选: Bug/功能/优化/文档]
├── 完成进度     [数字 0-100%]
├── 描述         [文本(多行)]
├── 附件         [附件]
├── 创建时间     [创建时间(自动)]
├── 修改时间     [修改时间(自动)]
└── 创建人       [创建人(自动)]
```

#### 推荐视图
| 视图 | 用途 | 配置 |
|------|------|------|
| 表格视图 | 全量浏览 | 按截止日期排序，分组=负责人 |
| 看板视图 | 状态流转 | 按状态分列，卡片显示名称+优先级+截止日期 |
| 日历视图 | 时间规划 | 按截止日期展示，一眼看排期 |
| 画廊视图 | 任务卡片 | 大卡片展示，含封面图+描述摘要 |

#### 自动化规则
```
规则1: 截止日期前1天 → 发送提醒给负责人
规则2: 状态变为"已完成" → 通知创建人
规则3: 优先级变为"紧急" → 发送消息给协作人
```

#### 关键仪表盘指标
- 任务总数 / 已完成 / 进行中 / 逾期
- 按负责人统计完成率
- 按优先级分布饼图
- 每周完成趋势折线图

### 场景2: 项目管理

#### 数据模型

```
项目表 (Projects)
├── 项目名称     [文本]
├── 状态         [单选: 规划中/进行中/暂停/已完成/已取消]
├── 负责人       [人员]
├── 团队成员     [人员(多选)]
├── 开始日期     [日期]
├── 截止日期     [日期]
├── 进度         [数字 0-100%]
├── 里程碑       [关联 → 里程碑表]
├── 任务列表     [关联 → 任务表]
├── 项目描述     [文本(多行)]
├── 预算         [数字(货币)]
└── 已使用预算   [公式: SUM(关联任务.费用)]

里程碑表 (Milestones)
├── 里程碑名称   [文本]
├── 所属项目     [关联 → 项目表]
├── 截止日期     [日期]
├── 状态         [单选: 未开始/进行中/已完成]
└── 备注         [文本]

任务表 (Tasks) — 复用任务管理模板，增加:
├── 所属项目     [关联 → 项目表]
├── 所属里程碑   [关联 → 里程碑表]
├── 预估工时     [数字(小时)]
├── 实际工时     [数字(小时)]
└── 工时偏差     [公式: 实际工时 - 预估工时]
```

#### 推荐视图
| 视图 | 用途 | 配置 |
|------|------|------|
| 甘特视图 | 时间线规划 | 按开始/截止日期绘制条形图，显示任务依赖关系 |
| 看板视图 | 状态总览 | 按项目状态分列 |
| 表格视图 | 详细编辑 | 分组=项目，排序=截止日期 |
| 表单视图 | 新建任务 | 表单收集任务信息，自动创建记录 |

#### 自动化规则
```
规则1: 项目进度达到100% → 通知所有团队成员
规则2: 里程碑逾期未完成 → 发送提醒给项目负责人
规则3: 新建任务 → 自动分配默认负责人
规则4: 每周一 9:00 → 发送项目周报摘要
```

#### 关键仪表盘指标
- 项目总览: 进行中/已完成/逾期数量
- 甘特图进度对比: 计划 vs 实际
- 资源负载: 每人并行任务数
- 预算消耗: 已用/总额

### 场景3: CRM 客户关系管理

#### 数据模型

```
客户表 (Customers)
├── 客户名称     [文本]
├── 联系人       [关联 → 联系人表]
├── 客户等级     [单选: A级/B级/C级/D级]
├── 行业         [单选: 互联网/制造业/金融/教育/零售/其他]
├── 地区         [单选: 华东/华南/华北/华中/西南/其他]
├── 来源         [单选: 官网/推荐/展会/广告/社交媒体]
├── 状态         [单选: 潜在客户/意向客户/成交客户/流失客户]
├── 首次接触日期 [日期]
├── 最近跟进日期 [日期]
├── 负责销售     [人员]
├── 累计成交额   [汇总: SUM(关联商机.金额)]
├── 商机数量     [汇总: COUNT(关联商机)]
├── 公司电话     [电话]
├── 公司邮箱     [邮箱]
├── 公司地址     [文本]
├── 备注         [文本(多行)]
└── 附件         [附件(合同/资质等)]

联系人表 (Contacts)
├── 姓名         [文本]
├── 所属客户     [关联 → 客户表]
├── 职位         [文本]
├── 手机         [电话]
├── 邮箱         [邮箱]
├── 微信         [文本]
├── 生日         [日期]
├── 角色         [单选: 决策者/影响者/使用者]
└── 备注         [文本]

商机表 (Opportunities)
├── 商机名称     [文本]
├── 所属客户     [关联 → 客户表]
├── 联系人       [关联 → 联系人表]
├── 阶段         [单选: 初步接触/需求确认/方案报价/商务谈判/签约成交/丢单]
├── 金额         [数字(货币)]
├── 预计成交日期 [日期]
├── 成交概率     [数字(百分比)]
├── 加权金额     [公式: 金额 × 成交概率]
├── 负责销售     [人员]
├── 产品         [多选]
├── 备注         [文本(多行)]
└── 丢单原因     [文本(仅阶段=丢单时显示)]

跟进记录表 (Activities)
├── 跟进主题     [文本]
├── 跟进类型     [单选: 电话/邮件/面访/微信/其他]
├── 所属客户     [关联 → 客户表]
├── 关联商机     [关联 → 商机表]
├── 跟进人       [人员]
├── 跟进时间     [日期]
├── 下次跟进时间 [日期]
└── 跟进内容     [文本(多行)]
```

#### 推荐视图
| 视图 | 用途 | 配置 |
|------|------|------|
| 看板视图(商机) | 销售漏斗 | 按阶段分列，卡片显示客户名+金额 |
| 表格视图(客户) | 客户列表 | 按等级分组，排序=最近跟进日期 |
| 表格视图(跟进) | 跟进日志 | 筛选=今天需跟进，排序=跟进时间 |
| 画廊视图(联系人) | 联系人卡片 | 显示头像+姓名+职位+公司 |
| 日历视图(跟进) | 拜访排期 | 按跟进时间展示 |

#### 自动化规则
```
规则1: 商机阶段变更 → 通知销售经理
规则2: 客户超过30天未跟进 → 发送提醒给负责销售
规则3: 联系人生日前7天 → 发送提醒
规则4: 商机成交 → 自动更新客户状态为"成交客户"
规则5: 新建客户 → 自动分配给对应地区销售
```

#### 关键仪表盘指标
- 销售漏斗: 各阶段商机数量和金额
- 月度销售额趋势折线图
- 销售业绩排行榜
- 客户转化率: 潜在 → 成交
- 跟进频率统计

## 三、模板的技术实现

### 模板数据结构

```typescript
interface Template {
  id: string;
  name: string;
  description: string;
  icon: string;
  category: string;           // 'task' | 'project' | 'crm' | 'hr' | 'inventory' | 'content' | ...
  industry?: string;          // '互联网' | '制造业' | '教育' | ...
  tags: string[];

  // 模板内容
  tables: TableTemplate[];    // 包含的表定义
  automations: Automation[];  // 预设自动化规则
  dashboards: Dashboard[];    // 预设仪表盘

  // 元信息
  author: string;
  version: string;
  sampleData: boolean;        // 是否包含示例数据
  thumbnail: string;          // 缩略图
}

interface TableTemplate {
  name: string;
  fields: FieldTemplate[];
  views: ViewTemplate[];
  defaultRecords?: Record[];  // 示例数据
}

interface FieldTemplate {
  name: string;
  type: string;               // 字段类型ID
  config: Record<string, any>; // 字段配置(选项列表、精度等)
  description?: string;
}

interface ViewTemplate {
  name: string;
  type: 'grid' | 'kanban' | 'gantt' | 'calendar' | 'gallery' | 'form';
  config: {
    groupBy?: string;         // 分组字段
    sortBy?: SortDef[];       // 排序
    filter?: FilterDSL;       // 筛选
    columns?: ColumnDef[];    // 列宽、顺序、隐藏
    stackBy?: string;         // 看板分组字段
  };
}
```

### 模板应用流程

```
1. 用户浏览模板中心 → 选择模板
2. 系统解析 Template JSON
3. 创建表: 遍历 tables → INSERT INTO tables + fields
4. 创建视图: 遍历 views → INSERT INTO views (含筛选/排序/分组配置)
5. 创建自动化: 遍历 automations → INSERT INTO automations
6. 创建仪表盘: 遍历 dashboards → INSERT INTO dashboards
7. (可选) 导入示例数据: INSERT INTO records
8. 完成 → 用户进入表格开始使用
```

### 模板市场设计

```
模板分三个层次:

1. 官方模板 (Official)
   - 产品团队维护，质量最高
   - 覆盖核心场景(任务/项目/CRM/HR/库存/内容)
   - 每个模板包含完整数据模型+视图+自动化+示例数据

2. 社区模板 (Community)
   - 用户创建并发布
   - 提交审核后上架
   - 覆盖长尾场景和行业特色需求

3. 企业模板 (Enterprise)
   - 针对企业定制
   - 可包含企业特有字段和流程
   - 支持批量分发到各部门
```

## 四、模板与系统的关系

```
┌─────────────────────────────────────────┐
│           模板中心 (Template Center)     │
│  官方模板 / 社区模板 / 企业模板          │
└──────────────┬──────────────────────────┘
               │ 选择模板
               ↓
┌─────────────────────────────────────────┐
│         模板引擎 (Template Engine)       │
│  解析 Template JSON → 创建数据结构       │
│  tables + fields + views + automations   │
│  + dashboards + sample data              │
└──────────────┬──────────────────────────┘
               │ 实例化
               ↓
┌─────────────────────────────────────────┐
│         用户工作区 (Workspace)            │
│                                          │
│  ┌──────────┐  ┌──────────┐             │
│  │ 任务管理  │  │ CRM系统  │  ...        │
│  │ 模板实例  │  │ 模板实例  │             │
│  └──────────┘  └──────────┘             │
│                                          │
│  用户可自由修改: 增删字段/视图/自动化     │
└─────────────────────────────────────────┘
```

关键: 模板是起点，不是终点。用户从模板开始，但可以自由修改所有内容。
