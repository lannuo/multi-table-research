## 二、模板的组成结构

### 2.1 Notion 模板组成

Notion 模板的本质是 **预构建的 Notion 页面（Prebuilt Page）**，可包含以下组件：

| 组件 | 说明 |
|------|------|
| **数据库（Database）** | 预定义的数据结构，含属性（Properties） |
| **视图（Views）** | 表格、看板、日历、时间线、画廊、列表等多种视图 |
| **属性（Properties）** | 文本、数字、日期、选择、关系（Relation）、公式（Formula）、汇总（Rollup）等 |
| **页面内容（Blocks）** | 标题、段落、Callout、Toggle、分割线等 |
| **数据库模板（Database Templates）** | 数据库内部的页面模板，可预设默认属性值 |
| **自动化（Automations）** | 数据库自动化，基于属性变更触发动作 |
| **关系（Relations）** | 多个数据库之间的关联 |

**自定义模板能力：**
- 修改页面封面、图标、标题
- 添加/修改文本格式（粗体、斜体、颜色等）
- 添加/删除 Block（Callout、Toggle、表格等）
- 修改数据库属性名称、类型、图标
- 添加新视图（看板、列表、日历、时间线、画廊）

### 2.2 Airtable 模板组成

Airtable 模板的本质是 **预构建的 Base（数据库容器）**，可包含以下组件：

| 组件 | 说明 |
|------|------|
| **数据表（Tables）** | 多个相互关联的数据表 |
| **字段（Fields）** | 文本、数字、日期、选择、关联记录（Linked Records）、查找（Lookup）、汇总（Rollup）、计数（Count）、公式（Formula）、附件、URL、条码等 |
| **视图（Views）** | 网格（Grid）、看板（Kanban）、日历（Calendar）、画廊（Gallery）、表单（Form）、甘特图（Gantt）、时间线（Timeline） |
| **自动化（Automations）** | 触发器 + 动作的工作流（如：记录匹配条件时触发 → 发送邮件/更新记录/运行脚本） |
| **接口/界面（Interfaces）** | 为不同用户角色定制的仪表盘和表单 |
| **扩展（Extensions）** | 脚本、图表、第三方集成等 |
| **示例数据（Sample Data）** | 预填充的记录，用于演示用法 |
| **颜色编码和分组** | 视觉组织设置 |

**特殊设计：**
- 模板自带示例数据，用户可通过 "Restart from scratch" 清除
- 支持按表清除数据（Clear data）
- Airtable Universe 发布时可选是否包含扩展

### 2.3 飞书多维表格模板组成

飞书多维表格模板是一个完整的 **业务系统包**，包含以下组件：

| 组件 | 说明 |
|------|------|
| **数据表（Datasheet）** | 多个数据表，支持跨表关联 |
| **字段（Fields）** | 文本、数字、日期、单选/多选、附件、人员、关联（单向/双向）、查找引用、公式、流程、按钮、自动编号、进度、评分、货币、地理位置、条码等 |
| **视图（Views）** | 表格视图、看板视图、甘特视图、画册视图、日历视图、表单视图 |
| **自动化（Automations）** | 触发条件（记录变更、按钮点击、Webhook、定时等）+ 执行动作（发送消息、修改字段、发送邮件、创建日程/任务/群组、发送 HTTP 请求等） |
| **工作流（Workflows）** | 更强大的流程引擎，支持条件判断、循环、多分支、AI 分类、AI Agent 节点 |
| **仪表盘（Dashboard）** | 数据可视化看板，支持柱状图、折线图、饼图、透视表、漏斗图、词云、雷达图、NPS 图、指标卡、排行榜等 |
| **高级权限（Permissions）** | 角色权限、行列权限、预览权限 |
| **应用模式（App Mode）** | 更高级的封装形式，通过拖拽搭建面向终端用户的交互界面 |

**核心字段类型体系（丰富的行业化字段）：**

```
常规字段：文本、数字、日期、单选、多选、复选框、人员、附件、超链接
业务字段：流程、按钮、自动编号、电话号码、Email、地理位置、条码、进度、货币、评分
高级字段：单向关联、双向关联、查找引用
智能字段：公式、AI 字段捷径（含 DeepSeek R1）
```

**自动化能力矩阵：**

```
触发条件：记录变更、按钮点击、Webhook、定时触发、飞书消息触发
执行动作：发送消息、修改字段、发送邮件、创建日程/任务/群组、发送 HTTP 请求、AI 生成文本
```

### 2.4 APITable / Vika 模板组成

| 组件 | 说明 |
|------|------|
| **数据表（Datasheets）** | 多表关联的数据表 |
| **字段（Fields）** | 与 Airtable 类似的丰富字段类型 |
| **视图（Views）** | 网格、看板、画廊、表单、甘特图等 |
| **自动化（Automations）** | 机器人自动化，支持触发器 + 动作 |
| **BI 仪表盘（Dashboard）** | 数据可视化 |
| **表单（Forms）** | 自动生成的数据收集表单 |
| **API 接口** | 一键自动生成 API，100+ 开放接口 |
| **可嵌入页面** | 模板页面可嵌入到外部网站 |

**使用流程：**
1. 在模板中心选择模板
2. 点击「使用模板」
3. 选择存放位置
4. 选择是否使用模板自带的原始数据
5. 可复制模板链接分享给协作者

## 三、模板的创建、分享与应用机制

### 3.1 Notion

**创建方式：**
- 任何 Notion 页面都可以成为模板
- 通过 Share → Publish → Publish to web 发布
- 必须开启 "Allow duplicate as template" 选项
- 生成公开 URL 供他人复制

**提交流程：**
1. 访问 `notion.com/templates`
2. 点击 "Submit a template"
3. 填写表单：公开链接、模板名称、描述、类别
4. 一次只能提交一个模板，但无总数限制

**使用方式：**
1. 浏览模板市场（按关键词/类别/创作者搜索）
2. 点击 "View template" 预览
3. 点击 "Start with this template" 复制到工作区
4. 自由修改、定制

**分发渠道：**
- 官方模板市场（Template Gallery）
- Notion Builders 计划（创作者激励计划，可获得推广收益）
- 第三方网站（个人博客、模板市场如 Gumroad 等）

**用户自建与发布：**
- 完全开放，任何用户都可以创建并提交模板
- Notion Builders 计划：专属创作者社区，可通过推广 Notion 赚取收益（最高 $50/每次获批试用）

### 3.2 Airtable

**创建方式：**
- 官方模板由 Airtable 团队开发和管理
- 用户创建的模板通过 Airtable Universe 发布

**Universe 发布流程：**
1. 访问 Airtable Universe 主页
2. 点击 "Publish and manage"
3. 完成公开 Airtable Universe 个人资料
4. 选择 Base → 填写标题、描述、分类 → "Publish base"
5. 可选是否包含扩展（Show extensions in published base）

**更新已发布模板：**
1. 修改原始 Base
2. 在 Universe 中找到该 Base
3. 点击 "Update base" → "Publish base contents"
4. 可预览未发布的更改

**取消发布：**
- 在 Universe 中找到该 Base → "Unpublish" → 确认

**使用方式：**
1. 浏览模板市场或 Universe（按类别/关键词搜索）
2. 点击模板查看概述和可视化示例
3. 点击 "Use template" → 选择工作区 → "Add base"
4. 可 "Restart from scratch" 清除示例数据

**关键区别：**
- 官方模板：Airtable 团队开发和维护
- Universe 模板：用户创建并发布（社区驱动）

### 3.3 飞书多维表格

**创建方式：**
- 官方模板由飞书团队开发
- 用户可创建「自定义模板」：将云文档（含多维表格）转换为自定义模板，方便团队重复使用
- 应用模式：通过拖拽方式搭建更高级的业务系统

**应用模式发布机制：**
- 应用开发者可基于多维表格构建开箱即用的业务应用
- 通过应用市场实现**曝光、分发与商业变现**
- 支持发布应用模板、多维表格模板和插件

**使用方式：**
1. 访问模板中心（`feishu.cn/template/group/base`）
2. 浏览或搜索模板
3. 一键使用，模板加载到工作区
4. 可根据需要定制

**特色机制：**
- 应用模式：比模板更高级的封装，一个应用可关联多个多维表格
- 千人千面：不同角色看到不同界面
- 多端适配：桌面端和移动端
- 官方定制渠道：找不到合适模板可申请官方定制

### 3.4 APITable / Vika

**创建方式：**
- 方法一：工作目录 → "+新建" → "从模板创建"
- 方法二：从模板中心创建
- 可创建「空间模板」：选择文件夹或文件 → 多功能按钮 → 创建模板

**使用方式：**
1. 打开模板中心
2. 选择想要的模板
3. 点击「使用模板」
4. 选择存放位置 + 是否使用模板自带数据
5. 复制模板链接分享给协作者

**分发渠道：**
- 模板中心（Template Center）
- 模板链接分享

## 五、核心发现与设计启示

### 5.1 模板的本质

各产品的模板本质是对 **预配置数据结构** 的封装，包含：
- **数据模型层**：表、字段、关联关系
- **展示层**：多种视图（表格、看板、甘特、日历等）
- **逻辑层**：自动化规则、工作流
- **分析层**：仪表盘、图表
- **权限层**：角色权限、行列权限

### 5.2 模板分类的最佳实践

从四个产品来看，模板分类通常有两个维度：
1. **按行业分类**（电商、制造业、教育、医疗等）—— 解决「我是谁」的问题
2. **按场景/功能分类**（项目管理、CRM、HR 等）—— 解决「我要做什么」的问题

飞书和 Airtable 还额外支持按「功能特性」分类（自动化、视图、AI 能力）。

### 5.3 模板生态的三个层次

1. **官方模板**：由产品团队维护，质量最高，覆盖核心场景
2. **社区模板**：由用户创建并发布到公开市场，数量庞大、场景丰富
3. **企业定制**：针对特定企业需求的定制化模板服务

### 5.4 飞书「应用模式」的创新

飞书的多维表格应用模式是一个值得关注的创新点：
- **超越模板**：不是简单地复制一个表格，而是封装成一个完整的「应用」
- **一个应用关联多个多维表格**：打破了单表模板的局限
- **千人千面**：不同角色看到不同界面
- **商业变现通道**：开发者可通过应用市场获得收入

### 5.5 模板设计的关键要素

一个优质的模板应包含：
1. **清晰的问题定义**：解决什么业务问题
2. **完整的数据模型**：预定义的字段、表、关联
3. **多样的视图**：表格 + 看板 + 甘特等
4. **自动化流程**：减少重复操作
5. **数据看板**：关键指标的可视化
6. **示例数据**：帮助用户理解模板用法
7. **使用说明**：教程或引导
